    DeepSeek本地部署及API调用指南

    网上众多deepseek本地部署的教程均混淆了一个概念，将API调用归到了本地部署这一栏，这是不准确的。严格意义上说，本地部署应当指的是数据完全在本地硬件设备进行处理，无需进行联网操作。但网上的众多推文为博热度，大部分将API调用也归到了本地部署这一块。 因此，我们要先区分二者的区别，根据自身条件判断选择更适合自己的方式使用deepseek R1模型。

    ▍核心概念区分（可视化对比表）

    维度 本地部署 API调用
    硬件依赖性 需高性能GPU/NPU（如RTX3090↑） 任意能上网设备
    数据流向 完全本地闭环 数据上传第三方服务器
    响应速度 依赖配置 网络波动影响
    典型场景 数据隐私性极度敏感/定制模型训练 快速原型开发/轻量级应用集成
    成本结构 前期硬件投入高（最低约3万人民币） 按API调用量计费(100w token约20人民币)

    ▍选择本地部署 Or API调用？

    可以通过多种维度去衡量评估适合自己的方式。上述的表格其实已经考虑了从多种维度评估二者的优劣性。

    API指的是Application Programming Interface，应用程序接口，是已经预先定义的一些方法，用户可以通过API去使用厂商提供的功能。 就拿API最擅长的并行处理举例，厂商本身就具备极强的硬件条件，能够去轻松的处理来自用户的API调用请求。

    拿一个应用场景举例，API接口可以让我们在5分钟内，让DeepSeek R1模型同时阅读大量的文献，而不需要考虑算力问题。 而本地部署的大模型，受算力的限制，同一时间，可能只能处理一个问题，并且处理问题的速度受硬件条件制约，一个小时内可能仅能处理几篇文献。

    但本地部署的模型他的好处就在于，所有的数据，都在本地进行处理了，无需通过互联网进行传输。而API调用的模型，需要将数据Or文档上传至服务器进行处理，在一定程度，存在数据泄露的风险。

    ▍本地部署成本 V.S API调用成本

    本地部署成本

    本地部署对硬件有极高的要求，且因英伟达禁售算力显卡的政策，市面上能流通的算力卡都存在极高的溢价（黄牛卡）。但华为昇腾服务器用的是他自研的算力卡，国产算力卡，但价格依旧十分高昂。之前在小红书搜索过华为昇腾服务器报价，在60w人民币，租的话大约是8元每小时。

    当然，上述部署的模型，可以供大型多人项目进行使用，因此成本会十分高昂。事实上，如果只想部署一个私人的，只有几个人使用的，并行处理要求不高的模型的话，硬件设备的要求可以放宽很多。

    目前来看，最便宜的671b部署方案的成本个人估算在3-5w元。

    API调用成本

    现在主流的API提供厂商关于DeepSeek API的定价都大差不差，硬要换算，大约是2000个问题20块钱的报价，下图是阿里云百炼大模型关于DS R1的计费方式。

    可以看得出，如果没有项目隐私性要求，以及大批量处理需求的话，API调用的成本要远低于本地部署。

    ▍本地部署指南

    本地部署模型的硬件要求太高了，首先我们需要对网上所谓的1.5B，7B，14B，32B，70B，671B的R1模型进行讲解。 671B指的是模型拥有6710 亿个参数，官网提供的R1模型，以及当下所谓的满血版，指的就是671B的R1模型。通俗来讲，大模型的知识量与参数的个数相关关系。以同一个模型如R1举例，低参数不一定不厉害，但是高参数肯定是比低参数的模型要厉害得多的。

    DeepSeek开源，也只是将671B的模型进行开源，市面上所谓的1.5B，7B，14B，32B，70B都是蒸馏模型(Distilled Model)。是基于671B模型的输出，去特化在某种应用上的处理能力，将用不到的参数进行舍弃，以此来提升处理速度以及能够在低成本设备上运行的能力。

    事实上，就我个人使用体验来说。 32B以下的模型可以说是智障，他们也许在某类问题能够处理的很好，比如宽泛的问题，能给一个宽泛的答案，但一旦深入问题去分析，就无法得出好的结果。因此，只推荐使用32B以上的模型，但就算能处理问题，仍旧是不如671B的模型的结果来得好。
